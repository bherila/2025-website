# Markdown Renderer

A browser-based Markdown editor and renderer at `/tools/markdown`. Anyone can paste Markdown and toggle to a styled preview; authenticated users can save documents and share them by short-code URL.

## Highlights

- GitHub-flavored Markdown (tables, task lists, strikethrough, autolinks) via `remark-gfm`.
- Fenced code blocks rendered with [Shiki](https://shiki.style) syntax highlighting using dual `github-light` / `github-dark` themes.
- ` ```mermaid ` fences render as inline SVG diagrams.
- Inline code rendered as styled chips (no surrounding backticks).
- "Print / Save as PDF" command waits for async previews (Mermaid, Shiki) and web fonts to settle, then triggers the browser print dialog.
- Optional save & share: authenticated users get a 7-character short code under `/tools/markdown/s/{code}`.

## Routes

| Method | URL                              | Auth          | Description                                            |
|--------|----------------------------------|---------------|--------------------------------------------------------|
| GET    | `/tools/markdown`                | Public        | Empty editor; signed-in users see "Save & Share"       |
| GET    | `/tools/markdown/s/{code}`       | Public read   | Renders a saved document; owner sees "Update"          |
| POST   | `/tools/markdown/save`           | `web` + `auth`| Create a new shared document, returns short code + URL |
| PATCH  | `/tools/markdown/s/{code}`       | `web` + `auth`| Update title and/or content; owner only                |

Routes are declared in `routes/web.php` (GET) and `routes/api.php` (POST/PATCH). The controller is `app/Http/Controllers/MD/MarkdownRendererController.php`.

## Storage

| Table | `markdown_documents` |
|---|---|
| Model | `App\Models\MarkdownDocument` |
| Migration | `database/migrations/2026_05_21_183245_create_markdown_documents_table.php` |
| Factory | `Database\Factories\MarkdownDocumentFactory` |

Columns: `id`, `user_id` (nullable, FK on `users.id`, `nullOnDelete`), `short_code` (unique, 10), `title` (nullable, 120), `markdown_content` (`mediumText`), timestamps.

Short codes are generated by `App\Support\ShortCode::generate(...)`, with a uniqueness predicate that re-rolls on collision.

## Permissions

- **View**: anyone with the URL. The controller does not check ownership on `showByCode`.
- **Save & Share** (create new document): any authenticated user; the new document's `user_id` is set to `Auth::id()`.
- **Update existing document**: only when `Auth::id() === document.user_id`. `MarkdownRendererController::update` returns 403 otherwise. The UI shows "Update" for the owner and "Viewing shared document" for everyone else.

Anonymous documents (`user_id = null`) cannot be updated through the UI because the API requires `auth` middleware.

## Frontend

| Path | Purpose |
|---|---|
| `resources/views/tools/markdown.blade.php` | Blade entrypoint; mounts the React tree |
| `resources/js/components/markdown/MarkdownRendererPage.tsx` | Top-level page: editor textarea, Markdown / Preview tabs, save/print buttons |
| `resources/js/components/markdown/Preview.tsx` | `react-markdown` wrapper with custom renderers for `code`, `pre`, table elements |
| `resources/js/components/markdown/ShikiBlock.tsx` | Shiki-rendered fenced code block; falls back to plain `<pre>` if the language is unknown |
| `resources/js/components/markdown/MermaidBlock.tsx` | Mermaid renderer; pipes generated SVG through `sanitizeSvg.ts` before injecting |
| `resources/js/components/markdown/printExport.ts` | Print-prep pipeline: waits on Shiki/Mermaid registry, then images, then fonts |
| `resources/js/components/markdown/previewRenderRegistry.ts` | Tiny pending/settled registry the async blocks register against so print can wait |
| `resources/js/components/markdown/markdownApi.ts` | `fetch` wrappers for save / update; reads CSRF token from `<meta name="csrf-token">` |

The Blade view passes a single `<script id="markdown-renderer-initial-data" type="application/json">` blob with `{document, markdown, title, canEdit, authenticated}`. The React app reads that blob and never re-fetches the document.

## Rendering pipeline

`MarkdownRendererPage` debounces the textarea by 150 ms and rebuilds a fresh `PreviewRenderRegistry` per change. `Preview` mounts `react-markdown` with `skipHtml`, `remarkGfm`, and component overrides:

- `pre` → identity passthrough; the actual block container is emitted by `ShikiBlock` to avoid `<pre><pre>` nesting.
- `code`:
  - If `className` is `language-mermaid` → `MermaidBlock`.
  - Otherwise if `rawText.endsWith('\n')` (i.e. fenced code block) → `ShikiBlock`.
  - Otherwise → inline `<code>` chip with `bg-muted` padding.
- `table` / `thead` / `tbody` / `tr` / `th` / `td` → responsive scroll wrapper plus alignment from `align` or `style.textAlign`.

`ShikiBlock` and `MermaidBlock` both register a pending key with the `PreviewRenderRegistry`, then mark it settled when their async work completes (or fails). `printExport.prepareAndPrint` awaits `registry.waitUntilSettled()` (with a 10 s ceiling), then images, then `document.fonts.ready`, before calling `window.print()`.

## Print / Save as PDF

Print styles live in `resources/css/app.css` under `@media print`. They:

- hide the navbar, toolbar, editor pane, action buttons, and any `.no-print` element;
- expand the preview to full width with no shadow or rounded corners;
- force `print-color-adjust: exact` so Shiki colors survive;
- avoid breaking inside `<pre>` and Mermaid blocks (`break-inside: avoid`);
- force Shiki tokens onto their light-theme colors regardless of dark mode.

To save as PDF, click "Print / Save as PDF" and pick "Save as PDF" in the browser print dialog.

## Styling notes

The preview container is `prose prose-neutral dark:prose-invert` (Tailwind Typography). The default theme wraps inline `<code>` with literal backtick characters via `code::before { content: '`' }` / `code::after { content: '`' }`. We override that in `resources/css/app.css`:

```css
.markdown-preview code::before,
.markdown-preview code::after {
  content: none;
}
```

A regression test in `resources/js/components/markdown/__tests__/Preview.test.tsx` ("disables the prose backtick pseudo-elements on markdown-preview code") reads `app.css` and fails if the override is removed.

## Tests

Unit tests live in `resources/js/components/markdown/__tests__/`:

- `Preview.test.tsx` — GFM tables, Mermaid routing, Shiki routing (labelled and unlabelled fences), inline code, prose backtick override.
- `ShikiBlock.test.tsx` — token rendering, fallback path.
- `MermaidBlock.test.tsx` — SVG sanitization, registry interaction.
- `MarkdownRendererPage.test.tsx` — tab switching, save/update flows, print preparation.
- `previewRenderRegistry.test.ts` and `printExport.test.ts` — async settling behavior.
- `sanitizeSvg.test.ts` — SVG sanitizer policy.

Run them with:

```bash
pnpm exec jest resources/js/components/markdown/__tests__ --runInBand
```

There is no Playwright E2E suite for the markdown renderer; the JSDom tests plus the print/Shiki/Mermaid unit tests cover the rendering pipeline. For manual verification, the in-PR convention is to open `http://localhost:8001/tools/markdown` or `http://localhost:8001/tools/markdown/s/{code}` and confirm the preview visually.
