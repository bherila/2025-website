## Maintenance Notes

- Prefer adding a row to an existing index before creating a new top-level doc.
- Use lowercase kebab-case filenames for Markdown docs.
- Avoid changelog-style "recent updates" sections; git history and PRs already provide that timeline.
- When a doc mentions a route, file path, or command, keep it specific enough to search for and verify.
