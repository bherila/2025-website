# Parking Pickup Visual Harness

A URL-driven mode for capturing deterministic screenshots of the Parking Pickup scene. Future visual-art PRs should attach before/after pairs generated by this harness.

## Quick start

Run a local dev server on port 8000, then:

```bash
pnpm exec playwright install chromium
pnpm run test:e2e:parking-pickup-visual
```

Screenshots land in `test-results/parking-pickup/` (gitignored) and are also attached to the Playwright HTML report.

## URL API

The harness is activated only when `visualTest=1` is present. The query string is the public contract:

| Param           | Required | Values                | Effect                                                                 |
| --------------- | -------- | --------------------- | ---------------------------------------------------------------------- |
| `visualTest`    | yes      | `1`                   | Activates the harness. Any other value is a no-op.                     |
| `level`         | yes      | integer ≥ 1           | Loads that level directly via `generateLevel()`.                       |
| `seed`          | no       | any string            | Reserved for future use. `generateLevel(level)` is already deterministic. |
| `hud`           | no       | `compact` / `normal`  | `compact` (default) keeps the mobile stats overlay collapsed; `normal` initialises it expanded. |
| `reducedMotion` | no       | `1`                   | Reserved for future use. CSS animation-disable is already handled by Playwright's `use.animations`. |
| `colorblind`    | no       | `1` / `0`             | Forces colorblind mode on or off, overriding stored preference.        |

Example URLs:

```
/games/parking-pickup?visualTest=1&level=1&seed=parking-style-a&hud=compact&reducedMotion=1
/games/parking-pickup?visualTest=1&level=8&seed=parking-style-a&hud=compact&reducedMotion=1
/games/parking-pickup?visualTest=1&level=20&seed=parking-style-a&hud=compact&reducedMotion=1
```

## Behaviour in visualTest mode

When `visualTest=1`:

- The game **bypasses** `loadProgress()` and `loadLevelSnapshot()`.
- The game **does not write** `bwh.cars-game.progress.v2` or `bwh.cars-game.snapshot.v2`.
- The tutorial overlay is suppressed.
- The colorblind preference is forced from the URL when `colorblind` is present; otherwise the stored value is used and **not** written back to localStorage.
- `generateLevel(level)` runs with the deterministic per-level seed (`seedForLevel(level)`), so the same URL produces the same layout across runs.

The normal production path is **inert** unless `visualTest=1` is present. Removing the query param restores normal saved-progress behaviour.

## Readiness signal

Playwright waits on a global flag instead of polling:

```ts
declare global {
  interface Window {
    __PARKING_PICKUP_VISUAL_READY__?: boolean
    __PARKING_PICKUP_VISUAL_STATE__?: {
      frameCount: number
      level: number
      renderedAt: number
      seed: string | null
    }
  }
}
```

The marker flips to `true` after the third rendered frame, by which point:

- React has mounted the game.
- The requested level has been generated.
- The Three.js renderer has sized to the current viewport.
- The scene has rendered three full frames (camera fit + dynamic mesh setup have settled).

In the Playwright spec:

```ts
await page.waitForFunction(() => window.__PARKING_PICKUP_VISUAL_READY__ === true, undefined, {
  timeout: 30_000,
})
```

## Scenarios captured today

`tests/e2e/parking-pickup.visual.spec.ts` covers:

- `level-1` — tutorial composition, lowest density.
- `level-8` — mid-curve density.
- `level-20` — high density, near the car-count cap.

Each scenario runs in both Playwright projects (`chromium-desktop` and `chromium-mobile`), producing six PNGs per run.

## Adding a new scenario

Append to the `SCENARIOS` array in `tests/e2e/parking-pickup.visual.spec.ts`:

```ts
{
  name: 'level-42',
  url: '/games/parking-pickup?visualTest=1&level=42&seed=parking-style-a&hud=compact&reducedMotion=1',
},
```

Update the `scenarioLevel` helper at the bottom of the file if the level expectation matters.

## Notes for visual-art PRs

- Generate before/after pairs by running the harness on `main` and on your branch, then attach both sets of PNGs to the PR.
- Keep changes additive — do not alter game rules, scoring, car movement, or generation difficulty in a visual-art PR. Those changes belong in separate engine-focused PRs.
- The harness must remain safe for production: never change scene/game behaviour outside the `visualTest=1` gate.
